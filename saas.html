ChAImp SaaS

Software for needs the market forgot — or priced out.

A modern, mobile-first home for your PTA/PTO.

Everything an animal rescue runs on — in one place.

Web presence, volunteers, adoptions, surrenders, fundraising

Co-built with Bull Terrier Rescue, Inc., RescueCentral grew out of what an active rescue actually needs to run, day in and day out. Animal rescues juggle a public website, adoption applications, surrender intake, volunteer and foster coordination, and ongoing fundraising — usually across half a dozen disconnected tools. AI works as triage and decision support: pre-reading adoption applications, flagging concerns for a volunteer to look at, and queuing up the next action so nothing falls through the cracks.

Security

Security isn't a feature.
It's the foundation.

Every ChAImp SaaS product is built — and kept — to the security standards mission-driven organizations deserve. Donors trust you with their data. Your constituents trust you with theirs. We treat that trust as the design constraint, not an afterthought.

Our pledge: security is the first decision we make on every product — and the standard every release is measured against. Not a checkbox. Not a launch-week scramble. The foundation, full stop.

🛡️

Human review on every change — with Claude Code as an extra gate.

Code at ChAImp is Claude Code–assisted, but it never reaches production without a human reviewer signing off. A person reads every change, understands what it does, and decides whether it ships. On top of that, every pull request is also scanned by Anthropic's Claude Code security review — an automated static check that flags injection flaws, auth bypasses, unsafe deserialization, secret exposure, and broken access control. Findings are remediated by a human before merge — never quietly waived, never auto-passed.

Human sign-off required Automated SAST gate Pre-merge, every PR
🔐

Sign in with Google — 2FA by default

Authentication runs through Google OAuth by default — a de facto standard on the web, and the identity provider most of your team already uses. Microsoft OAuth is available based on business need for organizations standardized on Microsoft 365. Either way, your team signs in with an account they already secure with 2FA and recovery — no new password to reuse, no shared logins, no forgotten-password tickets. The most common cause of nonprofit breaches is a reused password. We close that door from day one.

🔎

Gitleaks credential scanning

All code is checked in to GitHub with Gitleaks running on every push and pull request. API keys, tokens, and credentials are caught before code can land — and rotated immediately if anything is ever flagged.

📡

Continuous threat monitoring

We watch for new vulnerabilities, dependency CVEs, and emerging threats against the libraries and platforms our products depend on. When something matters, we patch — and tell you what we did.

🗄️

Row-Level Security across every database

Tenant and user isolation is enforced at the database layer with Row-Level Security policies — not just in application code. Even a logic bug in the app can't expose one organization's data to another, because the database itself refuses the query.

🔑

Managed secrets in Azure Key Vault

Application secrets, service credentials, and signing keys live in Microsoft Azure Key Vault — never in code, never in shared spreadsheets, never in email. Access is least-privilege, rotated on a schedule, and revoked the moment it's no longer needed.

🧾

Auditable by design

Authentication events, admin actions, and data exports are logged with timestamps and actor IDs. If a board, auditor, or grantor ever asks who did what, when? — we can answer.

🏗️

Built on infrastructure we don't have to apologize for.

ChAImp SaaS runs on platforms trusted across the industry, not a homegrown stack. Databases run on Supabase (managed PostgreSQL) with automated platform-managed backups. Applications are hosted on Vercel with HTTPS, managed certificates, and a global edge network by default. Application secrets are held in Microsoft Azure Key Vault. Code is version-controlled on GitHub, sign-in goes through Google OAuth, and AI features are powered by Anthropic — with your data never used to train commercial models. Established infrastructure means your nonprofit's data sits behind the same operational security that protects thousands of production applications — maintained by dedicated platform teams, not by us alone.

Our SaaS Principles

Built for the sector,
not extracted from it.

Commercial nonprofit software too often treats the sector as a market to mine. ChAImp SaaS operates by a different set of rules.

Nonprofit-appropriate pricing

Pricing scales with what an organization can realistically pay — including free tiers where the mission justifies it. No enterprise pricing dressed up with a discount badge.

Co-developed, not assumed

Every product is shaped with at least one real partner nonprofit before it goes broader. Features come from observed need — not from a roadmap meeting.

Your data, your call

You own your data. You can export it. We don't train commercial models on it. Governance is a default, not an upsell.

No lock-in by design

Open formats, documented exports, and a graceful exit path. If a different tool fits you better later, we'll help you move — not hold your data hostage.

Hosted with care, or self-host

Use our managed hosting, or run it on your own infrastructure. Either way, the product behaves the same.

ChAImp retains the IP

So the product can keep serving the sector long after any single engagement ends. Co-build partners get preferential terms, not invoice surprises.
